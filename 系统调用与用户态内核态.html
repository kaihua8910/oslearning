用户态与内核态
用户态（User Mode）：应用程序运行状态
· 只能访问用户空间
· 受限的CPU指令集

内核态（Kernel Mode）：操作系统运行状态
· 可访问全部内存和硬件
· 可执行全部CPU指令

为什么需要两种状态？
保护系统核心不被用户程序破坏
用户态 → 内核态：
通过中断/异常/系统调用触发
内核态 → 用户态：
通过 iretd 指令返回
CPU特权级与保护机制
x86处理器支持 4级特权级（Ring 0-3）：

Ring 0（内核态）：最高权限，可执行任何指令
Ring 1/2：保留给驱动程序
Ring 3（用户态）：最低权限，限制指令

Linux只使用Ring 0和3
· 内核运行于Ring 0
· 用户程序运行于Ring 3
CS寄存器低2位 = CPL
当前特权级（Current Privilege Level）
控制对关键资源的访问权限

系统调用流程：
1. 用户程序调用libc函数
2. lib通过syscall触发软中断
3. 内核处理，切换到Ring 0
4. 执行系统调用服务例程
5. 返回用户态
常见系统调用类型
进程控制：fork, exec, exit, wait
文件操作：open, read, write, close
设备通信：ioctl, read, write
信息维护：getpid, alarm, sleep
通信：pipe, shmget, mmap
保护：chmod, umask, chown

Linux系统调用号：
每个系统调用有唯一编号（如read=0, write=1）
x86-64 Linux系统调用：
rax = 系统调用号
rdi, rsi, rdx, r10, r8, r9 = 参数
rax = 返回值
