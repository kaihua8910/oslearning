Buffer Cache & Page Cache in OS
磁盘缓存原理
Buffer Cache / Page Cache：
内存中缓存磁盘数据的区域，减少磁盘IO。

为什么需要缓存？
· 磁盘访问时间 ~10ms，内存 ~100ns
· 相差 100,000 倍！

工作原理：
· 首次读取：从磁盘读入缓存，返回
· 再次读取：直接从内存返回
· 写操作：先写缓存，延迟刷盘
缓存命中率 = 缓存命中次数 / 总访问次数
90%命中率 → 性能提升10倍

缓存置换策略
1. 写回（Write-back）
数据先写入缓存，标记"脏"，后台异步刷盘
优点：减少IO次数，缺点：断电可能丢数据

2. 写穿（Write-through）
同时写缓存和磁盘
优点：数据安全，缺点：IO次数多

3. 预读（Read-ahead）
读取时顺便加载相邻数据
脏页 = 被修改过的缓存页
需要刷盘才能释放
Linux 页缓存架构
Linux 统一缓存模型：

· Page Cache：以页(4KB)为单位缓存文件
· Buffer Cache：以块(512B-4KB)为单位缓存块设备
· 现代Linux：两者统一为Page Cache

关键结构：
· radix tree：快速查找缓存页
· pdflush/flush：后台刷新脏页
· dirty_ratio：脏页比例上限
Linux缓存策略：
内存够用 → 尽量缓存
内存紧张 → 回收缓存页
