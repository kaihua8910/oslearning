📊 基础 Clock vs 改进 Clock 对比
🕐 基础 Clock (NRU)
核心机制
仅使用 访问位 U（1位开销）
时钟指针顺时针循环扫描

置换规则
U=1 → 清零为0，给一次"机会"
U=0 → 直接淘汰该页面

缺点
不区分脏页，淘汰修改过的页
可能产生 额外磁盘写回
⏱️ 改进 Clock 算法
核心机制
使用 访问位 U + 修改位 D
按四级优先级置换

置换优先级
① U=0,D=0 最优（无需写回）
② U=0,D=1 写回后淘汰
③/④ U=1 先清零，下轮再判断

优势
大幅减少磁盘I/O写回操作
🆚 与 LRU 的对比
LRU
精确记录访问顺序
硬件实现开销 极高
软件模拟 性能差
基础 Clock
仅用1位U标记
硬件开销 极低
近似LRU效果
改进 Clock
U + D 共2位
开销 极低
减少磁盘写回 效率更高
📚 核心知识点
什么是 Clock 算法？
Clock 算法 = NRU（最近未使用）算法，是 LRU 的低成本替代方案。
将内存页面组织为环形，用循环扫描指针代替 LRU 的精确时间戳，硬件开销远低于 LRU。
访问位 U（Used）的作用
页面被 CPU 访问时，硬件自动将 U 置为 1。
指针扫描到该页时，若 U=1 则清零并跳过；若 U=0 则认为"最近未使用"，直接淘汰。
修改位 D（Dirty）的作用
页面内容被修改（写入）时，D 置为 1（脏页）。
脏页被淘汰时必须先写回磁盘，开销较大。改进Clock优先淘汰干净页（D=0），减少磁盘I/O。
改进Clock四级优先级
优先级1：U=0, D=0 → 最佳淘汰对象，无需写回
优先级2：U=0, D=1 → 需写回磁盘
优先级3：U=1, D=0 → 先清U，下轮候选
优先级4：U=1, D=1 → 先清U，下轮写回
