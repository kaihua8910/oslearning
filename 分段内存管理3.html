分页 vs 分段 核心对比
📮 分页内存管理
固定大小的页 → 无外部碎片
地址：页号 + 页内偏移
适合机器效率，有内部碎片
📦 分段内存管理
逻辑段，长度可变 → 无内部碎片
地址：段号 + 段内偏移
贴合程序逻辑，有外部碎片
📚 核心知识点
分段地址转换步骤
1. 逻辑地址 [S, W]
2. 用 S 查段表 → 得到基址、段长
3. 越界检查：W < 段长？
4. 物理地址 = 基址 + W
Segmentation Fault（段错误）
当偏移量 W ≥ 段长时触发，系统拒绝访问。
这是保护内存安全的核心机制，防止进程越界访问其他段的数据。
分段的优势
• 符合程序逻辑结构
• 便于保护（代码段只读）
• 便于共享（多进程共用代码段）
• 有外部碎片（缺点）
段页式内存管理
结合分页+分段优势：先按逻辑分段，再将每段划分为固定页，解决外部碎片问题，是现代OS的主流方案。
