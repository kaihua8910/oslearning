管程 Monitor 概念
管程是一种高级同步原语，由Hoare提出：

将共享资源的操作和同步机制封装在一起，形成一个监视器对象。

管程特性：
· 互斥访问：同一时刻只有一个进程在管程内
· 条件变量：用于进程等待/唤醒
· 封装同步逻辑：更易读、不易出错

Java synchronized 就是管程
管程 = 共享数据 + 操作 + 同步
monitor ProducerConsumer {
... // 共享变量
procedure put() { ... }
procedure get() { ... }
}
管程内部结构
入口队列（Entry Queue）
想进入管程但被阻塞的进程排隊等待

互斥锁（Mutual Exclusion）
保证同一时刻只有一个进程在管程内

条件变量（Condition Variables）
· wait()：阻塞自己，等待条件
· signal()：唤醒等待该条件的进程

注意：signal后该进程继续执行还是被唤醒进程执行，取决于实现（Hoare vs Mesa语义）
wait(cv) → 阻塞，释放互斥锁
signal(cv) → 唤醒等待的进程
（两种语义：立即释放 or 延迟释放）

管程 vs 信号量 对比
信号量（Semaphore）
· 低级原语，需手动管理
· PV操作分散在代码各处
· 容易出错（顺序错误、遗漏PV）
· 适用于简单同步

管程（Monitor）
· 高级抽象，封装同步
· 同步逻辑集中，易维护
· 互斥自动保证
· 适合复杂共享资源管理
信号量：分散式同步
管程：集中式同步
