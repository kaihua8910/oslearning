银行家算法核心规则

1
核心公式：Need = Max - Allocation
剩余需求 = 最大需求 - 已分配
2
请求合法性校验：
Request ≤ Need（申请不能超过剩余需求）
Request ≤ Available（申请不能超过可用资源）
3
安全性检查核心：查找安全序列
能找到安全序列 → 安全状态 → 允许分配
找不到安全序列 → 不安全状态 → 拒绝分配
4
核心目标：避免进程死锁，保证系统资源安全调度
