Interrupt & Exception Handling
中断概述
中断（Interrupt）：硬件异步事件，打断CPU正常执行。

中断类型：
· 硬件中断：外部设备发出的信号（键盘、鼠标、磁盘、网卡）
· 软件中断：执行特定指令（int n, syscall）
· 异常（Exception）：指令执行中的错误（除零、缺页）
· 陷阱（Trap）：主动陷入（系统调用）

中断的作用：
实现CPU与设备的并行工作，提高系统效率
中断 = 异步事件
异常 = 同步错误
陷阱 = 主动陷入

异常与陷阱的区别
异常（Fault）：由错误条件引起，可能被修复后继续执行
· 除零错误（#DE）
· 缺页异常（#PF）← 可修复！
· 段错误（#GP, #PF）

陷阱（Trap）：主动触发，执行后返回继续
· 系统调用（syscall, int 0x80）
· 调试断点（int 3）
· 溢出检查（into）
异常 → 可能返回
(缺页可修复)
陷阱 → 必定返回
(主动触发)
中断向量表
中断向量表（IVT）：
每个中断号对应一个处理程序的入口地址。

x86 实模式：
位于内存 0x00000~0x003FF，共256项
每项 4 字节 = (CS:IP)

IDT（保护模式）
可位于内存任意位置，每项8字节（64位描述符）
中断号 → 处理程序入口
int 0x80 → sys_call
int 13h → 磁盘BIOS
Page Fault → do_page_fault
