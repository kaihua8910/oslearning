Peterson 算法
Software Mutual Exclusion Algorithm
Peterson 算法原理
Peterson算法：1981年由Gary Peterson提出，是一种基于软件的互斥算法。

核心思想：
· 使用两个共享变量：
- flag[0], flag[1]：表示进程是否想进入临界区
- turn：表示应该谁进入

特点：
· 无需硬件原子指令
· 纯软件实现互斥
· 适用于两个进程的互斥
共享变量：
boolean flag[2];
int turn;

flag[i]=true 表示进程i想进入
Peterson 算法代码
进程 P0（i=0）：
flag[0] = true;
turn = 1;
while(flag[1] && turn==1);
// 临界区
flag[0] = false;

进程 P1（i=1）：
flag[1] = true;
turn = 0;
while(flag[0] && turn==0);
// 临界区
flag[1] = false;
关键：
1. 先表示"我想进入"
2. 再让"给你先"
3. 最后等待对方不想进或轮到自己

算法正确性分析
满足三大特性：

1. 互斥访问
两个进程不能同时在临界区内。

2. 前进性
如果没有进程在临界区，想进入的进程应当能进入。

3. 有限等待
进程等待进入临界区的时间是有限的。
优点：纯软件，无需硬件支持
缺点：只适用于2个进程
基于忙等(busy-wait)
